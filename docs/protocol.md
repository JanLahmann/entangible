# Entangible WebSocket protocol ‚Äî v1

> Single source of truth for the wire protocol between `qamposer-physical-host`
> and its browser clients, and for the in-process contract between the vision
> pipeline and the host. `shared/ws/messages.ts` (`@shared/ws`) is the canonical
> TS schema ‚Äî it mirrors the shapes here 1:1 and a vitest parity test guards the
> field names. All JSON field names are camelCase.

## Endpoints (single HTTPS origin, default `:8443`)

| Path         | Transport            | Purpose                                    |
|--------------|----------------------|--------------------------------------------|
| `/ws/state`  | WebSocket, JSON text | Server ‚Üí clients: circuit/detection/status |
| `/ws/frames` | WebSocket, binary    | Phone camera ‚Üí server JPEG frames (M4)     |

## Staff security ‚Äî the operator token

The booth has a single shared **operator token**, generated by the host on
first run and stored with the TLS material (`qamposer-physical token` prints it,
`--rotate` mints a new one). It gates the staff/control surfaces; viewer
surfaces stay fully open (no token, zero friction).

| Surface                                              | Gate                          |
|------------------------------------------------------|-------------------------------|
| `/ws/state` reads (circuit/detection/status/layout)  | **open** ‚Äî any client         |
| `/ws/state` `select_camera` / `select_mode` / `select_layout` / `select_noise` / `select_menu` / `serve` | operator `hello` (below) |
| `/ws/frames` (phone frame intake)                    | `?key=<token>` (else close `4403`) |
| `GET /debug/stream`, `GET /debug/snapshot.jpg` (MJPEG preview) | `?key=` or `X-Operator-Key` header (else `403`) |
| `GET /api/qr`                                        | `?key=` or `X-Operator-Key` header (else `403`) |
| `/api/dispatch/*` (QN4 ‚Äî dispatch state, arm/disarm, Home Connect setup) | `?key=` or `X-Operator-Key` header (else `403`); the OAuth callback is instead guarded by its single-use `state` param |
| `GET /api/visitor-qr` (view-only pocket QR)          | **open** ‚Äî embeds no token    |
| `/`, `/pocket`, `/api/info`, the `/debug` page shell | **open**                      |

Distribution: the staff cheat-sheet / host QR embeds the token so a scanned
`/pocket?‚Ä¶&role=camera&key=‚Ä¶` or `/debug?key=‚Ä¶` link authenticates with no
typing; `/debug` otherwise prompts and stores the key in `localStorage`
(`entangible.operator.key`).

## `/ws/state` ‚Äî server ‚Üí client messages

Every message is a JSON object with a `type` discriminator.

### `circuit` ‚Äî sent on every *stable* circuit change

```jsonc
{
  "type": "circuit",
  "seq": 42,                  // monotonically increasing per host process
  "circuit": { "qubits": 5, "gates": [ /* @qamposer/react Circuit + controlled gates */ ] },
  "qasm": "OPENQASM 2.0;\n...",
  "source": "camera"          // "camera" | "replay" | "push"
}
```

- `seq` increments only when the circuit actually changes (deep equality),
  which the stabilizer already guarantees.
- The **latest** `circuit` message is replayed verbatim to every new client
  immediately after connect (late-joiner catch-up), before any live traffic.
- Beyond the native `@qamposer/react` `GateType`s, gates may carry the controlled
  types the ‚óè modifier emits (`CY`/`CZ`/`CH`/`CS`/`CT`, and `CCX` with an extra
  `control2` field); see `docs/marker-ids.md`. `qasm` renders these natively
  (`cy`/`cz`/`ch`/`ccx`, and `cu1(œÄ/2)`/`cu1(œÄ/4)` for `CS`/`CT`).

### `detection` ‚Äî diagnostics, throttled to ‚â§ 5 Hz

```jsonc
{
  "type": "detection",
  "fps": 12.4,                          // pipeline throughput, smoothed
  "board": {
    "found": true,
    "corners": 4,                       // corner markers currently visible (0-4)
    "reprojectionErrorMm": 0.05         // null when board not found
  },
  "markers": [
    { "id": 10, "row": 0, "col": 0 },   // on-grid gate tile
    { "id": 22, "offGrid": true }       // detected but rejected by grid mapping
  ],
  "warnings": [
    { "code": "lone_control", "message": "...", "row": 1, "col": 3 }
  ]
}
```

- Corner markers (IDs 0‚Äì3) are **not** listed in `markers`.
- `warnings[].code` values come from the circuit builder (`lone_control`,
  `lone_target`, `cell_conflict`, `control_ambiguous`, ‚Ä¶); `row`/`col` optional.
- Latest `detection` also replayed on connect (may be stale; `fps: 0` signals
  a stopped pipeline).

### `status` ‚Äî sent on connect and on every change

```jsonc
{
  "type": "status",
  "camera":  { "kind": "replay", "name": "fixtures/bell-sequence", "connected": true },
  "backend": { "enabled": false, "healthy": false },
  "clients": 2                          // current /ws/state client count
}
```

### `served` ‚Äî a Quantina serve, broadcast to every client (additive)

```jsonc
{
  "type": "served",
  "seq": 7,                   // host-stamped serve counter, monotonic per process
  "packId": "cocktails",      // the menu pack that was active when served
  "outcomes": ["101"],        // 1 bitstring for single/subset, k for shots mode
  "shotSource": "ideal"       // "ideal" | "noisy" | "real" ‚Äî where the sample came from
}
```

- The host is the authority: it stamps `seq` and `packId` (the current
  `layout.menu`) and fans the serve out to ALL clients, so every screen ‚Äî
  kiosk and viewer phones alike ‚Äî reveals the same result in sync.
- `shotSource: "real"` marks a staff-entered bitstring measured on a visitor's
  own device/account (the real-hardware serve loop, docs/quantina.md).
- Clients resolve `outcomes` ‚Üí items via the shared menu-pack mapping
  (`shared/menu/`). The **latest** `served` is replayed to late joiners after
  `layout`.

### `hello_ack` ‚Äî server's reply to `hello` (additive)

```jsonc
{ "type": "hello_ack", "role": "viewer" }
// role: "viewer" | "operator" ‚Äî this socket's authenticated standing
```

Sent to the `hello`-ing socket only. `operator` when the `hello` carried a
matching operator `key`; otherwise `viewer`. Viewers may read all state but may
not send the `select_*` control messages.

## `/ws/state` ‚Äî client ‚Üí server messages

```jsonc
{ "type": "hello", "role": "display", "client": "booth-screen", "key": "<token>" }
// role: "display" | "debug" | "capture-ui" | "camera" | "operator"
// client: free-form label, optional
// key: operator token; send it with role "operator" (or "camera") to unlock the
//      select_* controls. Viewers omit it (or send any other role) and stay read-only.
```

```jsonc
{ "type": "select_camera", "kind": "cv2", "index": 0 }
// kind: "cv2" | "picamera2" | "push" | "replay"; index only for cv2
```

- `hello` is courtesy metadata (feeds `status.clients` labeling); the server
  must not require it. A connection becomes **operator** only when it is a staff
  role (`role === "operator"` or the pocket-camera `role === "camera"`) **and**
  `key` matches the host token (constant-time compare); otherwise it stays a
  viewer regardless of the claimed role. The server replies with `hello_ack`
  carrying the resulting standing.
- `select_camera` swaps the pipeline's frame source at runtime; the server
  answers with a fresh `status`. **Operator-only** ‚Äî a viewer's `select_camera`
  / `select_mode` / `select_layout` / `select_noise` / `select_menu` / `serve`
  is silently ignored (no error, no `status`).
- Unknown/malformed client messages are logged and ignored ‚Äî never fatal.

### `layout` ‚Äî panel/mode state (additive, booth-v2)

```jsonc
{
  "type": "layout",
  "mode": "composer",                    // "composer" | "golf" | "quantina" | "attract"
  "sidebar": "right",                    // "right" | "left"
  "panels": ["results", "state", "qasm"], // visible panels, in order (registry names)
  "wires": "compact",                    // "compact" | "all" ‚Äî displayed wire count
  "noise": "off",                        // "off"|"falcon"|"eagle"|"heron"|"nighthawk" ‚Äî booth noise preset
  "menu": null                           // active Quantina menu-pack id, or null (string | null)
}
```

Broadcast on change; latest replayed to new clients after `status`. Panel
registry (booth-v2): `results`, `state`, `qasm`, `qsphere` (post-upstream),
`scorecard` + `minicircuit` (golf), `menu` + `order` (quantina), `branding`,
`camera` (operator-key-gated live camera; kiosks only ‚Äî see below). Unknown
names are ignored by clients (forward-compatible). `camera` is valid in every
mode but in no mode's preset, so `select_mode`'s panel reset never enables it ‚Äî
staff opt in per session from `/debug`. It renders ONLY on a kiosk holding the
operator key (it shows the `/debug/stream` MJPEG); viewer phones never surface
it (the connected app hides all camera UI). `mode` gained
`quantina` with QN2; clients on older bundles ignore it (unknown mode ‚Üí their
default rendering). In quantina mode a `menu` of `null` means "pack not chosen
yet" ‚Äî clients fall back to the built-in `coffee` pack.

### Client ‚Üí server (additive, booth-v2)

```jsonc
{ "type": "select_mode", "mode": "golf" }
```

```jsonc
// partial select_layout allowed ‚Äî omitted fields keep their current value
{ "type": "select_layout", "sidebar": "left", "panels": ["results", "qasm"] }
```

```jsonc
{ "type": "select_noise", "preset": "heron" }
// preset: "off"|"falcon"|"eagle"|"heron"|"nighthawk" ‚Äî the booth-wide in-browser
// noise-model preset. Operator-only (silently ignored from viewers, like the
// other select_*); an unknown value is ignored. Updates layout.noise, persists
// to layout.toml, and triggers a layout broadcast (also in late-joiner replay).
```

```jsonc
{ "type": "select_menu", "pack": "cocktails" }
// pack: the menu-pack id to activate (lowercase [a-z0-9-]). Operator-only
// (silently ignored from viewers). The host validates FORMAT only ‚Äî it cannot
// know which packs a client bundles ‚Äî persists layout.menu to layout.toml and
// broadcasts the new layout; clients falling on an unknown id use `coffee`.
```

```jsonc
{ "type": "serve", "outcomes": ["101"], "shotSource": "ideal" }
// A Quantina serve, sent by the serving surface (a provisioned kiosk's touch
// button or the /debug serve card) ‚Äî the sampler runs where the simulation
// runs (the client); the host is the authority that stamps and fans out.
// Operator-only. outcomes: 1..20 bitstrings (1-5 chars of 0/1 each; >1 only
// in shots mode); shotSource: "ideal" | "noisy" | "real" (default "ideal").
// Ignored when layout.menu is null (no active pack ‚Üí nothing to serve).
// The host replies by broadcasting `served` (above) to every client.
```

`select_mode` / `select_layout` / `select_menu` persist to `layout.toml` and
trigger a `layout` broadcast. REST siblings:
`GET /api/layout` (current layout JSON), `GET /api/branding`
(`{name, logoUrl|null, qrTarget}` from `branding.toml`, defaults when absent).

## QR endpoints (HTTP, PNG)

- `GET /api/qr` ‚Äî **staff-gated** (operator token; ¬ßStaff security). Default
  `path` is `/pocket?connect=1&role=camera` (the camera role; `/pocket`
  redirects to `/`). Encodes `<scheme>://<lan-ip>:<port><path>&key=<token>` ‚Äî
  the scanning phone arrives already carrying the token so it can reach the
  camera role + `/ws/frames`. Never shown to visitors.
- `GET /api/visitor-qr` ‚Äî **open** (no token). Encodes
  `<scheme>://<lan-ip>:<port>/pocket?connect=1`; the pocket app auto-connects
  to this host's `/ws/state` as a read-only viewer (the visitor "follow along +
  take your circuit home" QR on the booth footer + attract screen). Embeds no
  credential ‚Äî a viewer needs none. `/pocket` preserves the `?connect=1` query
  across its trailing-slash redirect.

Both respond `image/png` with `Cache-Control: no-store` and an `X-Encoded-URL`
header carrying the encoded target (for the `/debug` card + tests).

## `/ws/frames` ‚Äî phone capture (M4; server side lands in M2)

- **Requires the operator token** as a `?key=<token>` query parameter (the
  capture page is opened from a staff QR that carries it). An unauthenticated
  socket is accepted and immediately closed with policy code `4403`.
- Binary messages: JPEG-encoded frames, most-recent-wins (the server keeps a
  single latest-frame slot ‚Äî `PushFrameSource`).
- Optional JSON text messages: `{ "type": "hello", "role": "capture" }`.
- Backpressure is the **client's** job (`bufferedAmount`); the server never
  queues more than one frame.

## Reconnect rules (client)

- Exponential backoff 0.5 s ‚Üí 8 s, jittered, forever.
- On (re)connect the server replays latest `circuit` + `detection` + `status`
  (+ `layout`, + the latest `served` when one exists).
- If a replayed `seq` is *lower* than the last seen, the host restarted:
  accept it and reset the local counter (never discard as stale).

## In-process contract: vision pipeline ‚Üí host

`qamposer_vision.pipeline.Pipeline` runs the loop in a worker thread and emits
via callbacks (called on the worker thread; the host bridges to asyncio with
`loop.call_soon_threadsafe`):

```python
Pipeline(
    source: FrameSource,
    board_config: BoardConfig | None = None,   # default: assets.toml
    on_circuit: Callable[[CircuitEvent], None],    # stable changes only
    on_detection: Callable[[DetectionEvent], None] # every processed frame; host throttles to 5 Hz
)
.start() / .stop()                              # idempotent
.swap_source(source: FrameSource)               # runtime camera switch
.latest_annotated() -> np.ndarray | None        # BGR frame for /debug MJPEG
```

- `CircuitEvent`: `circuit: dict` (Circuit JSON), `qasm: str`, `source: str`.
  (`seq` is assigned by the host, not the pipeline.)
- `DetectionEvent`: `fps: float`, `board_found: bool`, `corners: int`,
  `reprojection_error_mm: float | None`, `markers: list[MarkerObs]`
  (`id`, `row`, `col` | `off_grid`), `warnings: list[BuildWarning]`.
  Snake_case in Python; the host serializes to the camelCase JSON above.

## Versioning

Additive changes (new optional fields, new message types) do not bump the
version; clients must ignore unknown fields and unknown `type`s. Breaking
changes bump to v2 and get a `{"type": "hello"}` server counterpart first.
